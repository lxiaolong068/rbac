## 文档更新规范

1. 更新日志格式
```markdown
## [版本号] - 日期

### 新增
- 新增功能1
- 新增功能2

### 优化
- 优化内容1
- 优化内容2

### 修复
- 修复问题1
- 修复问题2
```

2. 提交信息格式
```
docs: <type>(<scope>): <description>

- type: 新增|优化|修复
- scope: api|database|config
- description: 简要说明
```

3. PR标题格式
```
docs(<scope>): <description>
```
